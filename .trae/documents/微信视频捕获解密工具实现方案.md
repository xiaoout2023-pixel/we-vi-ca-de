# 微信视频捕获解密工具实现方案

## 项目概述

开发一个全自动化的微信视频捕获解密工具，支持个人微信和企业微信，在微信内置播放器中播放视频时自动捕获、下载并解密视频。

## 核心需求分析

### 需求 1：代理配置不影响视频播放

**技术背景**：
- 视频在微信内置播放器中播放（非浏览器）
- 个人微信使用 WeChatAppEx.exe 进程播放视频
- 企业微信使用 WXWorkAppEx.exe 进程播放视频
- 这些进程通过 HTTPS/HTTP2 协议与腾讯服务器通信

**矛盾分析**：
- MITM 代理需要拦截 HTTPS 流量，这需要安装证书并设置代理
- 代理拦截不会阻止视频播放，只是作为中间人转发并解密流量
- mitmproxy 是**透明代理**，对微信来说是无感知的
- 视频仍然正常播放，同时我们获取到了视频 URL

**最终方案**：

采用**系统代理 + mitmproxy 自动拦截**方案：

1. **不修改系统代理设置**（避免影响其他应用）
2. **使用 mitmdump 作为独立代理进程**，监听 localhost:8080
3. **使用环境变量或命令行参数**为微信进程单独设置代理：
   ```bash
   # 方法 A：通过命令行启动微信并设置代理（如果微信支持）
   WeChat.exe --proxy-server=127.0.0.1:8080
   
   # 方法 B：使用 Proxifier 等工具为特定进程设置代理
   # 仅对 WeChatAppEx.exe 和 WXWorkAppEx.exe 生效
   ```

4. **或者**：临时设置系统代理，程序结束时自动恢复
   - 通过 Windows 注册表修改系统代理
   - 仅拦截微信相关域名（通过 mitmproxy 白名单）
   - 程序退出时自动恢复原始代理设置

### 需求 2：分片视频处理（保持开放）

**当前认知**：
- 知乎文章和 GitHub 项目都显示视频是**完整 MP4 格式**
- 用户提到抓包遇到过**分片 MP4**（非 m3u8/DASH）
- 具体分片格式需通过实际日志分析确定

**处理策略**：

```
1. 默认假设：完整 MP4（知乎/GitHub 验证）
2. 保留扩展：支持未知分片格式
3. 实际测试：通过用户抓包日志确定真实格式
4. 灵活适配：根据实际格式调整处理逻辑
```

**暂不引入 FFmpeg**，除非：
- 实际测试确认视频确实是分片格式
- 且需要合并分片
- 或需要音视频流分离处理

### 需求 3：全自动化封装

**三大模块完全自动化**：

1. **监听捕获模块**
   - 自动启动 mitmdump
   - 自动拦截视频 URL（从 WeChatAppEx/WXWorkAppEx 进程）
   - 自动识别视频类型
   - 无需人工操作

2. **密钥提取模块**
   - 自动拦截 API 响应获取 decode_key
   - 自动匹配视频 URL 和 decode_key
   - 确保 key 和 URL 来自同一次请求

3. **视频下载解密模块**
   - 自动下载加密视频
   - 自动解密（Isaac64 + XOR）
   - 自动验证解密结果
   - 输出到 output/ 目录

**无需 GUI，命令行全自动运行**

#### 步骤 2.1：mitmdump 管理器 (`src/proxy/proxy_manager.py`)

```python
class MitmProxyManager:
    def __init__(self, port=8080, addon_path=None):
        self.port = port
        self.addon_path = addon_path
        self.process = None
        self.original_proxy = None
    
    def start(self):
        # 启动 mitmdump（独立进程，不影响其他应用）
        cmd = [
            "mitmdump",
            "--listen-port", str(self.port),
            "--set", "flow_detail=0",
            "--set", "connection_strategy=lazy",
            "--set", "ssl_insecure=true",
            "-s", self.addon_path,
            "--quiet"
        ]
        self.process = subprocess.Popen(cmd, stdout=PIPE, stderr=PIPE)
        
        # 等待代理启动
        time.sleep(2)
        if self.process.poll() is not None:
            raise Exception("mitmdump 启动失败")
    
    def stop(self):
        if self.process:
            self.process.terminate()
            self.process.wait(timeout=5)
    
    def __enter__(self):
        self.start()
        return self
    
    def __exit__(self, *args):
        self.stop()
```

```python
class SystemProxyController:
    """通过 Windows 注册表控制系统代理"""
    
    def __init__(self):
        self.registry_path = r"Software\Microsoft\Windows\CurrentVersion\Internet Settings"
    
    def enable_proxy(self, proxy="127.0.0.1:8080"):
        # 保存原始代理设置
        self.original_proxy = self._get_current_proxy()
        
        # 设置新代理
        winreg.SetValueEx(key, "ProxyEnable", 0, winreg.REG_DWORD, 1)
        winreg.SetValueEx(key, "ProxyServer", 0, winreg.REG_SZ, proxy)
        
        # 通知系统代理已更改
        self._notify_proxy_change()
    
    def disable_proxy(self):
        # 恢复原始代理设置
        if self.original_proxy:
            self._set_proxy(self.original_proxy)
        else:
            self._disable_proxy()
    
    def _get_current_proxy(self):
        # 读取当前代理设置
        pass
    
    def _notify_proxy_change(self):
        # 调用 WinAPI 通知代理更改
        import ctypes
        ctypes.windll.wininet.InternetSetOptionW(
            0, 39, 0, 0  # INTERNET_OPTION_SETTINGS_CHANGED
        )
        ctypes.windll.wininet.InternetSetOptionW(
            0, 37, 0, 0  # INTERNET_OPTION_REFRESH
        )
```

```python
class VideoCaptureAddon:
    def __init__(self, capture_callback):
        self.capture_callback = capture_callback
        self.pending_videos = {}  # 存储待处理的视频信息
    
    def response(self, flow: http.HTTPFlow):
        # 1. 只处理微信进程的请求
        if not self._is_wechat_request(flow):
            return
        
        # 2. 拦截视频下载 URL
        if self._is_video_url(flow.request.url):
            video_info = {
                "url": flow.request.url,
                "cookie": flow.request.headers.get("Cookie", ""),
                "title": self._extract_title(flow),
                "timestamp": time.time()
            }
            self.capture_callback("video_url", video_info)
        
        # 3. 拦截 API 响应获取 decode_key
        if self._is_api_response(flow):
            try:
                response_data = json.loads(flow.response.text)
                decode_key = self._extract_decode_key(response_data)
                video_id = self._extract_video_id(response_data)
                
                if decode_key and video_id:
                    self.capture_callback("decode_key", {
                        "video_id": video_id,
                        "decode_key": decode_key
                    })
            except:
                pass
    
    def _is_wechat_request(self, flow):
        user_agent = flow.request.headers.get("User-Agent", "")
        return "WeChat" in user_agent or "MicroMessenger" in user_agent
    
    def _is_video_url(self, url):
        # 判断是否为视频下载 URL
        return "finder.video.qq.com" in url and "stodownload" in url
    
    def _is_api_response(self, flow):
        # 判断是否为视频详情 API
        return "channels.weixin.qq.com/web/api/feed" in flow.request.url
```

#### 步骤 3.2：会话管理 (`src/capture/session_manager.py`)

```python
class SessionManager:
    """管理视频 URL 和 decode_key 的匹配关系"""
    
    def __init__(self):
        self.videos = {}  # {video_id: {"url": ..., "decode_key": ..., "cookie": ...}}
        self.ready_videos = []  # 已匹配完整的视频列表
    
    def add_video_url(self, url, cookie, title=None):
        # 从 URL 中提取 video_id
        video_id = self._extract_video_id_from_url(url)
        self.videos[video_id] = {
            "url": url,
            "cookie": cookie,
            "title": title or f"video_{int(time.time())}"
        }
        self._check_match(video_id)
    
    def add_decode_key(self, video_id, decode_key):
        if video_id in self.videos:
            self.videos[video_id]["decode_key"] = decode_key
            self._check_match(video_id)
    
    def _check_match(self, video_id):
        video = self.videos.get(video_id)
        if video and video.get("url") and video.get("decode_key"):
            self.ready_videos.append(video)
            self.capture_callback(video)
```

```python
class VideoDownloader:
    async def download(self, url, cookie, output_path):
        async with aiohttp.ClientSession() as session:
            async with session.get(url, headers={"Cookie": cookie}) as resp:
                if resp.status != 200:
                    raise Exception(f"下载失败：HTTP {resp.status}")
                
                total_size = int(resp.headers.get('content-length', 0))
                downloaded = 0
                
                with open(output_path, 'wb') as f:
                    async for chunk in resp.content.iter_chunked(8192):
                        f.write(chunk)
                        downloaded += len(chunk)
                        # 可选：显示进度
                
                return output_path
```

#### 步骤 5.1：Isaac64 算法实现 (`src/decode/isaac64.py`)

参考 GitHub 项目 Evil0ctal/WeChat-Channels-Video-File-Decryption 的实现。

**方案 A（推荐）：使用微信官方 WASM 模块**

```python
class Isaac64WASM:
    def __init__(self, wasm_path):
        self.wasm_path = wasm_path
        # 使用 wasmer-py 加载 WASM
        from wasmer import engine, Store, Module, Instance
        store = Store(engine.JIT())
        module = Module(store, open(wasm_path, 'rb').read())
        instance = Instance(module)
        self.instance = instance
    
    def generate_keystream(self, decode_key: int, length=131072) -> bytes:
        # 调用 WASM 函数生成密钥流
        result = self.instance.exports.generate(decode_key, length)
        
        # 关键：必须反转
        return bytes(result)[::-1]
```

#### 步骤 5.2：视频解密器 (`src/decode/video_decoder.py`)

```python
class VideoDecoder:
    def __init__(self, isaac64):
        self.isaac64 = isaac64
    
    def decrypt(self, encrypted_path, decode_key, output_path):
        # 1. 生成密钥流
        keystream = self.isaac64.generate_keystream(decode_key)
        
        # 2. 读取加密视频
        with open(encrypted_path, 'rb') as f:
            data = f.read()
        
        # 3. XOR 解密前 128KB
        decrypted_data = bytearray(data)
        for i in range(min(131072, len(data))):
            decrypted_data[i] = data[i] ^ keystream[i]
        
        # 4. 写入解密文件
        with open(output_path, 'wb') as f:
            f.write(decrypted_data)
        
        # 5. 验证解密结果
        self._verify_decryption(output_path)
        return output_path
    
    def _verify_decryption(self, file_path):
        with open(file_path, 'rb') as f:
            header = f.read(32)
            if b'ftyp' not in header:
                raise Exception("解密失败：未找到 ftyp 签名")
```

## 关键技术细节

### 1. 代理方案对比

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| 临时系统代理 | 简单可靠，所有流量都经过代理 | 影响其他应用，需手动恢复 | ⭐⭐⭐ |
| PAC 文件 | 可选择性代理 | 微信内置播放器可能不支持 PAC | ⭐⭐ |
| 进程级代理 | 精确控制 | 实现复杂，需第三方工具 | ⭐⭐⭐⭐ |
| 环境变量注入 | 轻量级 | 微信可能不支持 | ⭐⭐ |

**推荐方案**：临时系统代理 + 自动恢复
- 程序启动时设置系统代理
- mitmproxy 仅处理微信相关域名（通过白名单）
- 程序退出时自动恢复原始代理
- 对视频播放无影响（代理只是转发）

### 3. FFmpeg 是否需要？

**根据 GitHub 项目分析**：
- Evil0ctal 的项目**没有使用 FFmpeg**
- 仅需 Isaac64 算法 + XOR 解密
- 解密后直接输出 MP4

**结论**：
- **暂不引入 FFmpeg**
- 除非实际测试确认需要处理特殊格式
- 保持工具轻量级

### 4. Cookie 管理

- Cookie 有效期：12 小时（过 24:00 也会失效）
- 解决方案：
  - 捕获时保存最新 Cookie
  - 下载时使用最近捕获的 Cookie
  - 如 Cookie 失效，重新捕获

### 5. decode_key 获取方式

**方式 A（推荐）：拦截 API 响应**
- 从 API 响应中提取 decode_key
- 确保 URL 和 key 匹配

**方式 B（备用）：JS 注入**
- 修改微信返回的 JS 文件
- 仅在方式 A 失效时使用
