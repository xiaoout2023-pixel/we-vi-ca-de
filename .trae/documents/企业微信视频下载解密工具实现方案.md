# 企业微信视频下载解密工具实现方案

## 项目概述

开发一个全自动化微信视频捕获解密工具，支持**个人微信**和**企业微信**，实现：

1. 独立代理配置，不影响其他应用和视频播放
2. 自动拦截、下载、解密完整视频（支持分片视频自动合并）
3. 全流程自动化，无需人工干预

## 核心需求分析

### 需求 1：独立代理，不影响其他应用

**方案 A（推荐）：进程级代理注入**

* 使用 Windows 环境变量或工具（如 Proxifier）仅为微信进程设置代理

* 个人微信：`WeChat.exe`, `WeChatAppEx.exe`

* 企业微信：`WXWork.exe`, `WXWorkAppEx.exe`

**方案 B：PAC 文件智能路由**

* 创建自定义 PAC 文件，仅将微信域名路由到 mitmproxy

* 其他流量直接连接，不受影响

**方案 C：指定网卡代理**

* 为微信设置单独的网络路由规则

### 需求 2：处理分片视频

**视频格式识别**：

* **完整 MP4**：直接下载解密

* **HLS (m3u8)**：需要合并所有 .ts 分片

* **DASH**：需要合并视频流和音频流

**分片视频处理流程**：

1. 拦截 m3u8 播放列表 URL
2. 下载所有 .ts 分片
3. 使用 FFmpeg 合并为完整 MP4
4. 对合并后的视频进行解密

### 需求 3：全自动化

**自动化流程**：

```
启动程序 → 自动配置代理 → 后台监听 → 自动捕获视频 → 自动下载 → 自动解密 → 输出结果
```

**封装的工具**：

* 代理管理器：自动启动/配置/关闭 mitmproxy

* 视频捕获器：自动识别和收集视频信息

* 下载器：自动下载完整/分片视频

* 解密器：自动解密视频

* 无需 GUI，全部命令行或后台服务

```python
    pac_content = """
    function FindProxyForURL(url, host) {
        // 仅微信相关域名使用代理
        var wechat_domains = [
            "finder.video.qq.com",
            "channels.weixin.qq.com",
```

```python
class MitmProxyManager:
    def __init__(self, port=8080, addon_path=None):
        self.port = port
        self.addon_path = addon_path
        self.process = None
    
    def start(self):
        # 启动 mitmdump
        cmd = [
            "mitmdump",
            "--listen-port", str(self.port),
            "--set", "flow_detail=0",
            "--set", "connection_strategy=lazy",
            "-s", self.addon_path,
            "--quiet"
        ]
        self.process = subprocess.Popen(cmd, ...)
    
    def stop(self):
        # 优雅关闭代理
        if self.process:
            self.process.terminate()
            self.process.wait()
    
    def __enter__(self):
        self.start()
        return self
    
    def __exit__(self, ...):
        self.stop()
```

```python
class HLSMerger:
    async def download_and_merge(self, m3u8_url, cookie, output_path):
        # 1. 下载 m3u8 播放列表
        playlist = await self._download_playlist(m3u8_url, cookie)
        
        # 2. 解析所有 .ts 分片 URL
        ts_urls = self._parse_ts_urls(playlist, m3u8_url)
        
        # 3. 并发下载所有分片
        temp_dir = tempfile.mkdtemp()
        ts_files = await self._download_all_ts(ts_urls, cookie, temp_dir)
        
        # 4. 合并分片为完整 MP4
        await self._merge_ts_to_mp4(ts_files, output_path)
        
        # 5. 清理临时文件
        shutil.rmtree(temp_dir)
    
    async def _merge_ts_to_mp4(self, ts_files, output_path):
        # 使用 FFmpeg 合并
        ffmpeg.input('concat:' + '|'.join(ts_files)).output(
            output_path, 
            c='copy'
        ).run()
```

```python
# 使用微信官方 WASM 模块（最可靠）
class Isaac64WASM:
    def __init__(self, wasm_path):
        self.wasm_path = wasm_path
        # 使用 wasmer-py 或 pyodide 加载 WASM
        self.engine = self._load_wasm()
    
    def generate_keystream(self, decode_key: int, length=131072) -> bytes:
        """
        生成密钥流
        注意：必须 reverse() 反转
        """
        keystream = self.engine.generate(decode_key, length)
        return keystream[::-1]  # 关键：反转
```

#### 步骤 5.3：视频解密器 (`src/decode/video_decoder.py`)

```python
class VideoDecoder:
    def decrypt(self, encrypted_path, decode_key, output_path):
        # 1. 生成密钥流
        keystream = self.isaac64.generate_keystream(decode_key)
        
        # 2. 读取加密视频
        with open(encrypted_path, 'rb') as f:
            data = f.read()
        
        # 3. XOR 解密前 128KB
        decrypted_data = bytearray(data)
        for i in range(min(131072, len(data))):
            decrypted_data[i] = data[i] ^ keystream[i]
        
        # 4. 写入解密文件
        with open(output_path, 'wb') as f:
            f.write(decrypted_data)
        
        # 5. 验证解密结果
        self._verify_decryption(output_path)
    
    def _verify_decryption(self, file_path):
        # 检查文件头是否包含 ftyp 签名
        with open(file_path, 'rb') as f:
            header = f.read(32)
            assert b'ftyp' in header, "解密失败：未找到 ftyp 签名"
```

## 关键技术细节

### 1. 独立代理实现方案对比

| 方案     | 优点            | 缺点           | 推荐度   |
| ------ | ------------- | ------------ | ----- |
| PAC 文件 | 系统级支持，不影响其他应用 | 需要配置浏览器      | ⭐⭐⭐⭐⭐ |
| 进程注入   | 精确控制          | 实现复杂，需要管理员权限 | ⭐⭐⭐⭐  |
| 虚拟网卡   | 完全隔离          | 需要额外软件       | ⭐⭐⭐   |

**推荐方案**：PAC 文件 + 系统代理
